## Требования

### R1. Исследование и контент-стратегия
Провести исследование этапов генеалогического поиска, актуальных для Российской Федерации, и структурировать информацию для новичков:
- Начальные этапы (интервью с родственниками, разбор домашнего архива, систематизация данных).
- Организация структуры генеалогического исследования (как правильно организовать папки, файлы, фотографии, оцифрованные документы и источники на компьютере, чтобы не запутаться).
- Запросы в ЗАГС: правила подтверждения родства, список необходимых документов, пошлины.
- Запросы в региональные и федеральные архивы (государственные архивы субъектов РФ, военные архивы ЦАМО, РГВИА, РГАДА и др.).
- Создание шаблонов заявлений (запросы в ЗАГС, запросы в архивы, тематические и генеалогические запросы).

### R2. Шаблоны документов (Текст + Интерактивные формы)
Сайт должен содержать инструменты для работы с документами:
- Возможность просмотра и быстрого копирования простых текстовых шаблонов документов (с плейсхолдерами вроде [ФИО]).
- Отдельную интерактивную страницу или форму, где пользователь может заполнить свои данные (ФИО, адреса, реквизиты разыскиваемого лица и т.д.) и скачать сгенерированный документ в форматах PDF и DOCX на выбор.
- Шаблон структуры генеалогического исследования для скачивания (демо-проект): готовый ZIP-архив с эталонной структурой папок и текстовым файлом-инструкцией по ведению проекта, который новичок может скачать и использовать как основу на своем компьютере.

### R3. Технологический стек и дизайн
- Веб-сайт должен быть реализован на чистом HTML, CSS (Vanilla) и JavaScript без использования сборщиков (Vite, Webpack), фреймворков (React, Vue) или npm-пакетов (все внешние библиотеки, например, для генерации PDF/DOCX, должны подключаться через CDN).
- Премиальный дизайн: современный внешний вид (гармоничная цветовая палитра, качественная типографика, плавные анимации и ховер-эффекты), адаптивность под мобильные устройства и планшеты.
- Логическая структура страниц и навигация сайта должны быть сформированы на основе результатов проведенного исследования (вводная часть, организация проекта, шаги поиска, архивные запросы, интерактивные шаблоны).
- Полностью статический сайт, который корректно открывается напрямую через index.html или с помощью простейшего локального статического сервера.

## Критерии приемки

### Контент и структура
- Наличие пошагового руководства от старта (семейный архив, опрос родственников) до глубоких исследований (ЗАГС, госархивы, военные архивы).
- Подробные рекомендации по правильной организации и структурированию файлов и папок исследования на компьютере.
- Наличие не менее 4 различных шаблонов документов, актуальных для России (например, заявление в ЗАГС о выдаче справки о рождении/смерти, запрос в ЦАМО, общий генеалогический запрос в региональный архив).

### Шаблоны и интерактивность
- Раздел с просмотром и копированием текстовых шаблонов.
- Интерактивная форма ввода данных для каждого из 4+ документов.
- Кнопки генерации и скачивания заполненного документа в формате PDF (с сохранением форматирования).
- Кнопки генерации и скачивания заполненного документа в формате DOCX.
- Наличие кнопки/ссылки на скачивание ZIP-архива с готовым демо-шаблоном структуры генеалогического исследования (пустые структурированные папки с файлом README.txt).

### Дизайн и качество кода
- Адаптивная верстка (мобильные устройства, планшеты, десктопы).
- Премиальное визуальное оформление (отсутствие дефолтных стилей браузера, плавная анимация, проработанный UI/UX).
- Использование исключительно чистого HTML/CSS/JS (внешние JS-скрипты для PDF/DOCX подключены через CDN).
- Отсутствие ошибок в консоли браузера при работе с интерактивными формами.
