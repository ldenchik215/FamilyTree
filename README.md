# 📂 Родословная.Поиск — Генеалогический поиск в России

Веб-приложение и пошаговое руководство для начинающих исследователей, помогающее восстановить историю семьи в России. Проект объединяет методические рекомендации, готовую структуру хранения архива на ПК и интерактивный конструктор официальных запросов в государственные органы (ЗАГС, архивы, ЦАМО).

---

## 🌟 Ключевые возможности

1. **📖 Интерактивное пошаговое руководство (`guide.html`)**
   * Разбор домашнего архива, правила сканирования и оцифровки документов.
   * Методология опроса родственников (с готовыми опросниками).
   * Подробные инструкции по работе с ЗАГС и архивами (федеральными, региональными, военными).
   
2. **📁 Стандартизация семейного архива на ПК (`pc_structure.html`)**
   * Готовая **эталонная структура папок** для хранения сканов, фотографий и отчетов.
   * Шаблон с автоматической сортировкой файлов на компьютере.
   * Возможность скачивания готового ZIP-архива структуры (`FamilyTree_DemoStructure.zip`) с файлом-инструкцией `README.txt`.
   * Правила стандартизированного именования файлов для поддержания порядка в исследовании.

3. **✏️ Интерактивный конструктор документов (`templates.html`)**
   * Просмотр и копирование **4 профессиональных шаблонов** официальных запросов.
   * Генерация документов прямо в браузере через интерактивную веб-форму.
   * Экспорт готовых заявлений в форматы **PDF** (с сохранением оригинальной верстки) и **Word (DOCX)** для печати и отправки.
   
4. **🌐 Справочник полезных ресурсов (`resources.html`)**
   * Ссылки на ключевые базы данных (Память народа, gwar.mil.ru, ОБД Мемориал, ВГД, Familio).
   * Интерактивная база крупнейших федеральных и региональных архивов РФ с контактными данными и адресами.

5. **🧬 ДНК-генеалогия и методология**
   * Обзор видов ДНК-тестов (аутосомный, Y-ДНК, mtДНК) и принципов их интеграции в классический архивный поиск.
   * Описание шкалы уровней доказанности родства (Предание ➔ Гипотеза ➔ Вероятно ➔ Подтверждено).

## 🛠️ Технологический стек

Проект реализован с соблюдением принципов легковесности и независимости от внешней инфраструктуры:
* **Интерфейс и логика**: Чистый **HTML5**, **CSS3 (Vanilla)** и современный **JavaScript (ES6+)**.
* **Архитектура**: Полностью статический сайт, работающий без сборщиков (Webpack, Vite), Node.js или npm-зависимостей.
* **Генерация документов**: Библиотеки для экспорта документов подключены через надежные CDN-сети:
  * `docx.js` — для динамического формирования файлов Word (DOCX).
  * `html2pdf.js` — для экспорта экранной формы в PDF.
  * `FileSaver.js` — для безопасного скачивания файлов на стороне клиента.

## 🚀 Как запустить проект локально

Сайт полностью автономен и не требует установки зависимостей или компиляции. Вы можете запустить его следующими способами:

### Способ 1: Прямой запуск в браузере
Просто дважды кликните по файлу `index.html` в проводнике вашего компьютера. Страницы и скрипты будут работать корректно (включая генерацию документов и скачивание ZIP-архива).

### Способ 2: Запуск через локальный веб-сервер (рекомендуется)
Если вам нужен полноценный HTTP-сервер для тестирования или демонстрации:

* **Используя Python** (если установлен):
  Откройте терминал в папке проекта и выполните команду:
  ```bash
  python -m http.server 8000
  ```
  После этого откройте в браузере адрес: [http://localhost:8000](http://localhost:8000)

* **Используя Node.js** (пакет `http-server`):
  ```bash
  npx http-server -p 8000
  ```
  После этого откройте в браузере адрес: [http://localhost:8000](http://localhost:8000)

* **Используя VS Code**:
  Установите расширение **Live Server**, нажмите кнопку **Go Live** в правом нижнем углу окна редактора.

## 📝 Методические рекомендации для пользователей

### 1. Правило именования файлов на ПК
Для избежания путаницы все оцифрованные файлы домашнего архива и ответы на запросы следует переименовывать по единому шаблону:
`ГГГГММДД_[Фамилия]_[Имя]_[ТипДокумента]_[Описание]`

*Примеры:*
* `19430512_Иванов_Иван_СправкаОСмерти_ЗАГС.pdf`
* `19120824_Петрова_Мария_МетрикаРождение_ПриходскаяЦерковь.jpg`
* `19411205_Иванов_Петр_НаграднойЛист_ПамятьНарода.pdf`

### 2. Приоритет при запросах в ЗАГС
При составлении запросов о событиях после 1918–1926 гг. всегда запрашивайте **Справку с иными сведениями** (а не просто повторное Свидетельство). В графу «Иные сведения» сотрудники ЗАГС вносят дополнительные записи из архивной книги: точный адрес, место работы, причину смерти, состав заявителей и т.д., что является бесценным источником информации для поиска.

### 3. Шкала статусов родства в древе
* 🟥 **Предание** — устный рассказ родственников без документальных подтверждений.
* 🟧 **Гипотеза** — логичная версия на базе косвенных фактов или частичных совпадений.
* 🟨 **Вероятно** — версия подтверждается несколькими независимыми косвенными документами.
* 🟩 **Подтверждено** — родственная связь доказана прямым архивным документом (свидетельством или записью в метрике).
